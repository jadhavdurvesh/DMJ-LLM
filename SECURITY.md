# 🔒 Security Policy

Thank you for helping keep **DMJ LLM** secure.

The security of our users, contributors, and community is a priority. If you discover a security vulnerability, please report it responsibly so it can be investigated and resolved before public disclosure.

---

# Supported Versions

The following table indicates which versions currently receive security updates.

| Version | Supported |
|----------|:---------:|
| Latest Development | ✅ |
| Older Versions | ❌ |

As the project evolves, additional release branches may receive security updates.

---

# Reporting a Vulnerability

Please **do not** open a public GitHub Issue for security vulnerabilities.

Instead, report vulnerabilities privately through one of the following methods:

- GitHub Security Advisories (preferred)
- Project security email (when available)

If neither option is available, please contact the project maintainers before making the issue public.

---

# What to Include

Please provide as much information as possible, including:

- Description of the vulnerability
- Steps to reproduce
- Affected component(s)
- Potential impact
- Screenshots or logs (if applicable)
- Suggested mitigation (optional)

The more detail you provide, the faster we can investigate.

---

# Response Process

After receiving a report, the maintainers will:

1. Acknowledge receipt of the report.
2. Investigate the issue.
3. Determine the severity and impact.
4. Develop and test a fix.
5. Release the fix.
6. Publicly disclose the issue when appropriate.

---

# Responsible Disclosure

We kindly ask that you:

- Do not publicly disclose vulnerabilities before they are fixed.
- Allow maintainers reasonable time to investigate and resolve the issue.
- Avoid exploiting vulnerabilities beyond what is necessary to demonstrate the problem.

Responsible disclosure helps protect users and contributors.

---

# Scope

Examples of security-related issues include:

- Remote code execution
- Authentication or authorization bypass
- Dependency vulnerabilities
- Sensitive data exposure
- Supply chain attacks
- Injection vulnerabilities
- Denial-of-service vulnerabilities

General bugs and feature requests should be submitted through GitHub Issues instead.

---

# Security Updates

Security fixes will be included in future releases and documented in release notes when appropriate.

Users are encouraged to keep their local copies of DMJ LLM up to date.

---

# Contact

At present, the preferred reporting method is **GitHub Security Advisories**.

A dedicated project security email may be added in future releases.
